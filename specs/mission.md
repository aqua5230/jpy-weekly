# Mission

## 專案定位

本專案是「日圓週報自動化系統」。

它每週整合匯率、市場部位、央行資產負債表、國際收支、財政融資、技術面與短線消息，產生日圓投資參考報告。

核心判斷框架是 Werner 四原則。

## 使用者

- 主要使用者：需要追蹤日圓方向、干預風險與部位建議的投資者。
- 次要使用者：維護週報流程、判斷引擎、資料源與發送管線的開發者。

## 要解決的問題

- 每週手動整理日圓資料太耗時。
- 單一資料源容易失效，報告需要 fallback 與告警。
- 宏觀資料、技術面與短線消息容易混在一起，需要固定判斷框架。
- AI 產生內容可能污染報告，需要格式清理與合規檢查。
- 投資判斷需要可回測，不能只靠當週主觀感覺。

## 目前已完成能力

- 產生日圓週報主流程。
- 以 Werner 四原則輸出日圓方向、信心與支持/反對來源。
- 抓取 USD/JPY、EUR/JPY、COT、TFF、FRED、BoJ、MOF、行事曆與技術指標。
- 產出 Telegram 公開版、VIP 版、報告卡片、Telegraph 與 GitHub Pages HTML。
- 具備部分資料 fallback、快取與資料源告警。
- 具備 prediction log、1 週與 8 週回測、持有型回測。

## 暫時不做

- 不自動下單。
- 不提供保證獲利訊號。
- 不把短線 AI 觀察作為主判斷。
- 不把 Telegram、Telegraph 或 GitHub Pages 換成大型 CMS。
- 不在沒有 spec 的情況下做大型重構。

## 成功標準

- 每週可穩定產出報告。
- 主要資料源失效時，系統能降級、告警或使用快取。
- 報告判斷能追溯到 Werner 四原則與資料來源。
- 每次功能變更都有明確驗證方式。
- 下一個 AI session 可以只讀 `specs/`、`SESSION.md` 與相關程式碼就接續工作。
